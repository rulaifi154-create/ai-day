# AI-Day 活动网页 PRD

## 目标
搭建持续开放的 AI-Day 活动网页，支持登录、提交作品、投票，并引导用户参与。

## 范围
Web 端（桌面 + 移动 H5），Google 登录后访问全部页面和操作；活动与投票无限期开放，无开始/结束时间。

## 用户故事
- 作为已登录用户，我想查看活动介绍并导航到提交或投票页，以便快速参与。
- 作为已登录用户，我想提交作品并立即出现在投票列表中，以便他人可投票。
- 作为已登录用户，我想查看所有作品并用 3 票支持不同作品，以便表达偏好。

## 页面与功能

### 全局
- 访问控制：所有页面与操作需 Google 登录；未登录访问任意页面即跳转登录。
- 导航：显式入口到首页、作品提交页、我要投票页；显示当前用户信息与退出。

### 首页
- 内容：简要活动介绍文案、视觉横幅（可选）。
- 按钮：跳转到「作品提交页」与「我要投票页」。

### 作品提交页
- 表单：作品名称（必填）、作品简介（必填，长度限制），提交按钮。
- 行为：提交后即写入作品列表；成功反馈；表单重置。
- 校验：必填、长度限制、避免重复提交（按钮 loading/禁用态）。

### 我要投票页
- 列表：展示所有已提交作品（名称、简介、提交者、提交时间）。
- 投票规则：每个登录用户总计 3 票；每票需投给不同作品；投出后不可撤销或重复投同一作品；用尽 3 票后禁用投票交互。
- 状态反馈：剩余票数显示；投票成功/失败提示；投票中 loading 防重复操作。

## 数据与状态
- 用户：Google OAuth 获取用户标识；存用户 ID、姓名、头像（可选）。
- 作品：id、名称、简介、提交者、提交时间。
- 投票：投票记录含用户 id、作品 id、时间；服务端校验 3 票上限与唯一性。
- 列表更新：提交成功后作品即时出现在投票页；投票后票数即时刷新。

## 权限与安全
- 全站需登录；未登录操作均拒绝并引导登录。
- 服务端强校验投票规则（3 票、不可重复/取消）。
- 防刷：接口节流/去重；必要的错误提示。

## 非功能
- 可用性：移动端适配；按钮/表单状态清晰。
- 持续性：活动长期开放，无提交/投票截止时间，需确保长期可用性与数据维护策略（备份、清理异常数据）。
- 性能：列表加载支持分页或懒加载（若作品多时）。
- 可观测性：关键事件埋点（登录、提交、投票）。

## 验收标准
- 未登录访问任意页会触发 Google 登录，登录后返回原目标页。
- 登录用户能在首页看到介绍并跳转两目标页。
- 提交作品成功后，作品实时出现在投票页列表。
- 投票页展示全部作品；用户最多投 3 个不同作品，且无法撤销或对同一作品重复投票；票数与剩余额实时更新。
- 活动长期开放，无提交与投票截止时间；长期开放状态下仍保持稳定访问与数据一致性。
- 全流程有明确的成功/错误提示与加载状态。
